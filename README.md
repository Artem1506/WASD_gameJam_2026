## 🎮 Концепция и Игровой процесс

**«Зеленый Змей»** — это хардкорный survival-экшен на базе классической механики «Змейка», переосмысленной через геймдизайнерскую призму **нестабильного управления (Input Lag)**. 

### Основные механики:
* **Опьянение и лаг ввода**: Сбор испорченного винограда повышает уровень опьянения, что увеличивает задержку реакции змейки на команды WASD (возникает задержка ввода). Игроку приходится рассчитывать повороты заранее с учетом изменяющегося лага.
* **Экспоненциальное ускорение**: Физическая скорость змейки плавно возрастает по мере роста очков.
* **Механика Combo (Risk-Reward)**: Накопление комбо-очков за непрерывный сбор полезных ягод временно замораживает рост лага (комбо 5) или полностью обнуляет его (комбо 10).
* **Локальный мультиплеер (1v1)**: Соревновательный режим на одной арене со взаимными столкновениями хвостов и раздельным расчетом лага для каждого игрока.

### 🍎 Игровые предметы:
* **Свежий виноград (`spr_grapeFresh`)**: +1 очко к счету, +1 к комбо.
* **Испорченный виноград (`spr_grapeRotten`)**: +1 к уровню опьянения (увеличивает лаг), сброс комбо.
* **Таблетка (`spr_pill`)**: снижает уровень опьянения на 3 пункта (уменьшает лаг).
* **Защитная каска (`spr_helmet`)**: поглощает последствия одной аварии (блокирует рост лага и сброс комбо).
* **Мусорный бак (`spr_cleaner`)**: очищает 50% случайных испорченных ягод на поле.
* **Банан (`spr_banan`)**: +3 очка/комбо, но удваивает скорость движения на 3 секунды.

### ⌨️ Управление:
* **Игрок 1**: клавиши `W`, `A`, `S`, `D` для движения.
* **Игрок 2**: клавиши-стрелки.
* **Клавиша `Esc`**: пауза.

## 🛠 Технические особенности и оптимизация HTML5

Проект адаптирован для бесшовной работы как на ПК, так и в браузере (HTML5).

### 🖥️ Адаптивный ресайз (HTML5 Canvas Scaling)
* **Интеграция JS-расширения**: Подключено расширение `html5` (`extensions/html5/html5.yy`), внедряющее JavaScript-функции `evalJS` и `browser_stretch_canvas` для динамического ресайза холста в браузере.
* **Объект-контроллер `oResize`**: Размещен во всех комнатах игры (`rm_start`, `rm_main`, `rm_2Players`, `rm_result`). Он автоматически подстраивает размеры окна, камер, портов и GUI под размеры окна браузера при изменении его масштаба.
* **Кроссплатформенный GML-fallback**: Создан GML-скрипт `scripts/evalJS/evalJS.gml`. На Windows-сборках он перехватывает вызовы `evalJS` и возвращает размеры стандартного окна GameMaker, предотвращая ошибки компиляции ("unknown function") и падения на десктопе.

### ⏸️ Стабильность меню паузы и шейдеров
* **Сортировка слоев при рисовании**: Объект `obj_pause` динамически переопределяет глубину на `depth = 150` в событии `Create`, благодаря чему размытый скриншот игры (`pause_surf`) и черное перекрытие гарантированно отрисовываются позади кнопок интерфейса (`HUD`, глубина `100`), не закрывая их.
* **Защита от крашей WebGL при смене комнат**: Во избежание зависаний HTML5-рантайма при переходе из меню паузы в комнату результатов, в кнопке `obj_buttonToResult` перед вызовом `room_goto` принудительно восстанавливается активность всех деактивированных объектов (`instance_activate_all()`).
* **Корректные шейдеры**: Шейдеры размытия (`sh_blur`), затемнения (`sh_blackout`) и лага (`sh_inputLag`) приведены к WebGL-совместимому синтаксису (GLSL ES) с корректными объявлениями точности `precision`.

## 📁 Структура проекта

```text
sprites/   - графические ассеты и спрайты интерфейса
objects/   - управляющие контроллеры и интерактивные объекты
rooms/     - уровни игры (rm_start, rm_main, rm_2Players, rm_result)
audio/     - музыкальное сопровождение и звуковые эффекты
scripts/   - скрипты логики ввода, движения змейки и ресайза
shaders/   - графические шейдеры (размытие, лаг, blackout)
fonts/     - игровые шрифты (поддержка кириллицы)
```

## 📝 Правила нейминга ассетов

Каждый ассет в проекте должен иметь соответствующий префикс:

| Префикс | Назначение |
|---|---|
| `fn_`   | Шрифты |
| `obj_`  | Объекты |
| `ps_`   | Системы частиц |
| `pt_`   | Пути и кривые |
| `rm_`   | Комнаты / уровни |
| `sh_`   | Шейдеры |
| `snd_`  | Аудиофайлы |
| `spr_`  | Спрайты |
| `sc_`   | Скрипты |

## 💻 Кодстайл

| Элемент | Стиль | Пример |
|---|---|---|
| Скрипты / функции | camelCase | `playEatFresh` |
| Локальные переменные | snake_case с подчеркиванием | `_surf_w` |
| Объекты | obj_PascalCase | `obj_snakeHead` |
| Константы / Макросы | UPPER_SNAKE_CASE | `MAX_LAG` |
| Перечисления (enum) | PascalCase | `GameState` |
| Спрайты / Звуки | spr_camelCase / snd_camelCase | `spr_grapeFresh` / `snd_collision` |
| Шейдеры | sh_camelCase | `sh_blur` |

## 🐙 Git Workflow

* **Основная ветка**: `main`
* **Feature-ветки**: Новые задачи выполняются в отдельных ветках `feature/название-фичи` и вливаются через Pull Request.
* **Правило коммитов**: Коммиты должны иметь ясные, лаконичные англоязычные названия (например, `Fix pause menu button depth`, `Add oResize fallback for Windows`).
