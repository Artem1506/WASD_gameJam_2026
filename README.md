## Git Workflow

Главная ветка проекта: `main`

- Для каждой новой задачи создаётся отдельная feature-ветка.
- Слияние изменений выполняется через Pull Request.
- Перед началом работы необходимо обновить локальную версию проекта.
- Перед коммитом проект должен успешно запускаться без критических ошибок.

### Формат названий веток

```text
feature/название-фичи
```

---

## Правила работы команды

- Все важные изменения фиксируются в Git.
- Не удалять чужие ассеты без согласования.
- Крупные изменения обсуждаются заранее.
- Коммиты должны иметь понятные названия.
- Не загружать временные или неиспользуемые файлы в репозиторий.

### Примеры хороших коммитов

```text
Add player jump mechanic
Fix collision bug
Update UI layout
```

---

## Правила нейминга ассетов

Каждый ассет должен иметь префикс, обозначающий его тип.

| Префикс | Назначение |
|---|---|
| `fn_`   | Шрифты |
| `obj_`  | Объекты |
| `ps_`   | Системы частиц |
| `pt_`   | Пути и кривые |
| `rm_`   | Комнаты / уровни |
| `sh_`   | Шейдеры |
| `snd_`  | Аудиофайлы |
| `spr_`  | Спрайты |
| `sc_`   | Скрипты |

## Структура проекта

